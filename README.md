# 🌾 お米 注文・精米・発送 管理システム

BASE と LINE の注文を集約し、**今日精米すべき量**を表示。発送はワンクリックで
**ヤマトB2クラウド用CSV**を作成し、そのままアップロード→送り状印刷ができます。

## 使い方（毎日の流れ）

1. **`run.bat` をダブルクリック** → ブラウザでアプリが開きます
2. **🛒 BASE取込** … BASEの注文を取り込む（CSV または API）
3. **📝 注文入力** … LINEで来た注文を、お客様を選んで追加（新規のみ住所入力）
4. **🌾 ダッシュボード** … 今日精米する量を確認して精米作業
5. **🚚 ヤマト出力** … 発送する注文を選んでCSV作成
   → ヤマトB2クラウドの「送り状発行 → 外部データ取込」にアップロード → 印刷
   → 発送できたら「出荷済みにする」を押す（ダッシュボードから自動で外れます）

## 画面

| ページ | 役割 |
|--------|------|
| 🌾 ダッシュボード | ハブ画面。精米量＋**顧客別の発送リスト(誰に何を何kg)**＋取込/ヤマト/出荷ボタン |
| 📝 注文入力 | LINE/コメフル等の注文をクリックで追加。既存顧客は選ぶだけ、新規は入力 |
| 🚚 ヤマト出力 | 送り状CSV(B2クラウド形式)を作成し、注文を出荷済みにする |
| 🛒 取込 | BASE(CSV/API)とコメフル(CSV)の注文を取り込む |
| ⚙️ 設定 | 送り主・商品マスタ・顧客・BASE API・データ管理 |

### デザイン・操作性
- 阿部農園／琥珀米のブランドカラー（クリーム×藍×琥珀ゴールド、明朝見出し）
- **スマホ対応**（ボタン全幅・カード縦積み）。スマホのブラウザからも操作可

### CSVの出力先
- 「ヤマトCSVを保存」を押すと、**デスクトップの『ヤマト出荷CSV』フォルダ**に
  `YYYYMMDD_ヤマト出荷用出力データ.csv` というファイル名で保存されます（無ければ自動作成）。
- 手元に控えが欲しいときは「CSVをダウンロード（控え）」も使えます。

### 予約出力（PC起動時の自動書き出し）
- PCのフォルダに今すぐ書けないときは、出力を**予約**としてためておき、
  PCが起動すると自動で『ヤマト出荷CSV』に書き出します。
- `スタートアップ登録.bat` を一度実行すると、PC起動のたびに自動処理されます
  （解除は `スタートアップ解除.bat`）。`agent.py` がその処理本体です。
- ※ 「PC停止中にスマホから予約 → 起動時に自動保存」を完全に動かすには、
  クラウド公開（スマホとPCでデータ共有）が必要です。仕組みは実装済みです。

### 出荷ボタンと各サイト連携
- **BASE**：API設定済みなら「出荷完了」で自動的に発送完了（`orders/edit_status`）にします
- **コメフル**：公開APIが無いため、「出荷完了」時に出店者管理画面リンクを表示（手動処理）
- いずれも自社システムでは出荷済みになり、ダッシュボードの精米量から外れます

## 仕組みのポイント

- 顧客マスタは、最初に置かれていた発行済データCSVから自動生成済み（39名）
- 送り状CSVは**元データと同じ97列・Shift-JIS**なので、B2クラウドにそのまま取込可能
- 品名の全角/半角ゆれ（精米５㎏ / 精米5㎏ など）は自動で名寄せして集計
- 「精米が必要」な商品だけが精米量に加算（玄米・やさいは別枠、複合は精米kg入力待ち）

## 構成（開発者向け）

```
Home.py                 ダッシュボード（トップ）
pages/                  各画面
lib/db.py               SQLite データ層
lib/logic.py            品名正規化・精米量集計・初期商品マスタ
lib/yamato.py           ヤマト送り状CSV 読み書き（CP932・97列）
lib/yamato_header.py    送り状CSVの列定義
lib/seed.py             発行済データCSV → 顧客マスタ
lib/base_api.py         BASE 取込（CSV / API）
lib/bootstrap.py        起動時初期化
data/app.db             データ本体（SQLite）
.streamlit/config.toml  画面テーマ
```

セットアップ（再構築する場合）:

```powershell
py -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe -m streamlit run Home.py
```
